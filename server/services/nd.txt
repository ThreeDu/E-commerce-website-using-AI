PHAN TICH CHI TIET DU AN (Senior Software Engineer)
===================================================

Muc tieu: phan tich tung thanh phan trong repo, neu ro luong xu ly, business logic, API, DB va AI pipeline.

1) TONG QUAN HE THONG
* Bai toan: e-commerce co AI chatbot va dich vu ML cho Customer Intelligence (churn + potential score).
* Kien truc: React SPA (frontend) -> Node.js/Express API (backend) -> MongoDB (data) + Python Flask ML service (AI).
* Flow chinh:
  * Frontend gui request -> Express routes -> controllers -> models (MongoDB) -> response JSON.
  * Chatbot: UI -> /api/chatbot/message -> chatbotService -> (rule + LLM + retrieval + ranking) -> goi y san pham.
  * Chatbot + ML: UI -> /api/chatbot/message -> chatbotService -> fetch ML score (/api/intelligence/customer/:id) -> mlBoost trong ranking -> reply.
  * ML: /api/intelligence/* tren ml-service -> feature engineering tu MongoDB -> model predict -> tra JSON.
* Cong nghe:
  * Frontend: React, react-router-dom, Context API.
  * Backend: Node.js, Express, Mongoose, JWT, bcrypt.
  * ML: Flask, scikit-learn, pandas, numpy, joblib, pymongo.
* Mo hinh: Layered MVC (routes -> controllers -> services -> models). ML service tach rieng.

3.3 Services (AI/business)
* server/services/recommendationService.js
  * getPersonalizedRecommendations(userId, limit):
    * Lay AnalyticsEvent (product_view/add_to_cart), Orders, wishlist
    * Tao categoryHints/brandHints
    * Query Product theo category/brand, sort theo totalPurchases/ratings/views
    * Fallback trending neu thieu data
  * getAnonymousRecommendations(anonymousId):
    * Dung AnalyticsEvent theo anonymousId
  * getTrendingProducts: sort theo totalPurchases/totalViews/averageRating

* server/chatbot-service/service.js (quan trong nhat)
  * Session memory in-memory: sessionStore Map, MAX_HISTORY = 10, timeout 30 phut.
  * Short-term memory: luu budget/category/brand/model trong session.
  * LLM query parser: maybeParseQueryWithLlm
    * Ho tro OpenAI chat/completions va Gemini generateContent.
    * Tra ve JSON {intent, brand, series, model, product_line, storage, ram, price_min, price_max, confidence}
  * Budget parsing: parseBudgetFromText + parsePriceConstraint
    * Nhận dang: "32.000.000", "20 tr", "500k", co xu ly "gia bao nhieu".
  * Intent detection: greeting, policy, product_consult, product_search.
  * Behavior profile: tu context.userBehavior (viewed/cart/preferredCategory).
  * KNN co-occurrence (chatbot events):
    * refreshKnnCache: query ChatbotEvent 7 ngay, build matrix product-actor.
    * computeKnnScores: cosine similarity tu event weights.
  * LTR ranking: rankWithLtr
    * Features: semanticFeature, knnScore, behaviorFeature, popularity, rating, priceFit
    * Weighting khac nhau neu query la budget-driven.
  * Retrieval + ranking pipeline: findRecommendedProducts
    * Filter theo category/price/LLM hints
    * Strict name filter neu exact search
    * Tinh ltrScore + hardMatchOverride
    * Dedup, threshold ltrScore
  * LLM reply: maybeGenerateLlmReply
    * Context: top 4 products + recent history
    * Fallback rule reply neu LLM khong hop le
    * Timeout theo CHATBOT_LLM_TIMEOUT_MS (mac dinh 2500ms)
  * ML personalization:
    * goi ML service /api/intelligence/customer/:id de lay churn/potential
    * them mlBoost nhe trong LTR (discount/priceFit hoac rating/premium)
  * Response:
    * { sessionId, intent, reply, replyStructured, products[], quickReplies[], metadata }

7) AI/CHATBOT PIPELINE (CHI TIET)
* Intent detection: rule-based tu normalizeText + keyword.
* Query parsing:
  * priceConstraint: parsePriceConstraint (min/max, strictUpperBound)
  * LLM query hints: maybeParseQueryWithLlm -> JSON hints + confidence
* Retrieval/candidate:
  * Product.find(stock > 0, limit 400)
  * Filter theo category/price/LLM hints/strict name tokens
* Ranking:
  * semanticFeature = token overlap (name/category/desc)
  * behaviorFeature = viewed/cart/preferredCategories
  * knnScore = cosine similarity tu chatbot events
  * popularityFeature = log10(totalPurchases, totalViews)
  * priceFitFeature = khoang cach budget
  * LTR score = weighted sum (2 branch: budget vs semantic)
* LLM response:
  * Build prompt (intent + products + history)
  * Validate LLM output (nep danh sach san pham)
* Fallback rule-based: buildRuleReply + quick replies.
